# 编写自定义提供商

AI SDK 提供了一个[语言模型规范](https://github.com/vercel/ai/tree/main/packages/provider/src/language-model/v2)，使您能够创建与 AI SDK 兼容的自定义提供商。此规范确保不同提供商之间的一致性。

## 发布您的提供商

请将您的自定义提供商发布到您自己的 GitHub 仓库并作为 NPM 包。您负责托管和维护您的提供商。发布后，您可以向 AI SDK 仓库提交 PR，将您的提供商添加到[社区提供商](/providers/community-providers)文档部分。请使用 [OpenRouter 提供商文档](https://github.com/vercel/ai/blob/main/content/providers/03-community-providers/13-openrouter.mdx)作为您文档的模板。

## 为什么需要语言模型规范？

语言模型规范 V3 是一个用于与语言模型交互的标准化规范，为所有 AI 提供商提供了一个统一的抽象层。此规范创建了一个一致的接口，可与不同的语言模型无缝协作，确保开发人员可以使用相同的模式和方法与任何提供商交互。它支持：

<Note>
  如果您开源了一个提供商，我们很乐意在这里推广它。请向我们发送一个
  PR，将其添加到[社区提供商](/providers/community-providers)部分。
</Note>

## 理解 V3 规范

语言模型规范 V3 创建了一个健壮的抽象层，适用于所有当前和未来的 AI 提供商。通过建立标准化接口，它提供了支持新兴 LLM 能力的灵活性，同时确保您的应用程序代码保持与提供商无关并面向未来。

### 架构

V2 规范的核心定义了三个主要接口：

1.  **ProviderV3**：作为不同模型类型工厂的顶级接口
2.  **LanguageModelV3**：文本生成模型的主要接口
3.  **EmbeddingModelV3** 和 **ImageModelV3**：嵌入和图像生成的接口

### `ProviderV3`

`ProviderV3` 接口作为入口点：

```ts
interface ProviderV3 {
  languageModel(modelId: string): LanguageModelV3;
  embeddingModel(modelId: string): EmbeddingModelV3<string>;
  imageModel(modelId: string): ImageModelV3;
}
```

### `LanguageModelV3`

`LanguageModelV3` 接口定义了您的提供商必须实现的方法：

```ts
interface LanguageModelV3 {
  specificationVersion: 'V3';
  provider: string;
  modelId: string;
  supportedUrls: Record<string, RegExp[]>;

  doGenerate(options: LanguageModelV3CallOptions): Promise<GenerateResult>;
  doStream(options: LanguageModelV3CallOptions): Promise<StreamResult>;
}
```

关键方面：

- **specificationVersion**：必须为 'V3'
- **supportedUrls**：声明提供商可以本地处理哪些 URL（用于文件部分）
- **doGenerate/doStream**：用于非流式和流式生成的方法

### 理解输入与输出

在深入了解细节之前，理解 V3 规范中两个关键概念之间的区别很重要：

1.  **LanguageModelV3Content**：模型生成内容的规范
2.  **LanguageModelV3Prompt**：发送给模型的输入规范

### `LanguageModelV3Content`

V3 规范支持五种不同的模型可以生成的内容类型，每种都设计用于特定用例：

#### 文本内容

所有文本生成的基本构建块：

```ts
type LanguageModelV3Text = {
  type: 'text';
  text: string;
};
```

这用于标准模型响应、系统消息和任何纯文本输出。

#### 工具调用

使模型能够使用结构化参数调用函数：

```ts
type LanguageModelV3ToolCall = {
  type: 'tool-call';
  toolCallType: 'function';
  toolCallId: string;
  toolName: string;
  args: string;
};
```

`toolCallId` 对于将工具结果与其调用关联起来至关重要，尤其是在流式场景中。

#### 文件生成

支持多模态输出生成：

```ts
type LanguageModelV3File = {
  type: 'file';
  mediaType: string; // IANA 媒体类型（例如 'image/png', 'audio/mp3'）
  data: string | Uint8Array; // 生成的文件数据，作为 base64 编码字符串或二进制数据
};
```

这使得模型能够直接生成图像、音频、文档和其他文件类型。

#### 推理

专门支持思维链推理（对于像 OpenAI 的 o1 这样的模型至关重要）：

```ts
type LanguageModelV3Reasoning = {
  type: 'reasoning';
  text: string;

  /**
   * 可选的提供商特定元数据，用于推理部分。
   */
  providerMetadata?: SharedV2ProviderMetadata;
};
```

推理内容与常规文本分开跟踪，允许正确的 token 计数和 UI 呈现。

#### 来源

```ts
type LanguageModelV3Source = {
  type: 'source';
  sourceType: 'url';
  id: string;
  url: string;
  title?: string;
  providerMetadata?: SharedV2ProviderMetadata;
};
```

### `LanguageModelV3Prompt`

V3 提示格式 (`LanguageModelV3Prompt`) 被设计为一个灵活的消息数组，支持多模态输入：

#### 消息角色

每条消息都有一个特定的角色，允许特定的内容类型：

- **System**：模型指令（仅文本）

  ```ts
  { role: 'system', content: string }
  ```

- **User**：支持文本和文件的人类输入

  ```ts
  { role: 'user', content: Array<LanguageModelV3TextPart | LanguageModelV3FilePart> }
  ```

- **Assistant**：模型输出，支持完整的内容类型

  ```ts
  { role: 'assistant', content: Array<LanguageModelV3TextPart | LanguageModelV3FilePart | LanguageModelV3ReasoningPart | LanguageModelV3ToolCallPart> }
  ```

- **Tool**：工具执行的结果
  ```ts
  { role: 'tool', content: Array<LanguageModelV3ToolResultPart> }
  ```

#### 提示部分

提示部分是提示结构中消息的构建块。虽然 `LanguageModelV3Content` 表示模型的输出内容，但提示部分专门用于构建输入消息。每个消息角色支持不同类型的提示部分：

- **系统消息**：仅支持文本内容
- **用户消息**：支持文本和文件部分
- **助手消息**：支持文本、文件、推理和工具调用部分
- **工具消息**：仅支持工具结果部分

让我们探讨每种提示部分类型：

##### 文本部分

最基本的提示部分，包含纯文本内容：

```ts
interface LanguageModelV3TextPart {
  type: 'text';
  text: string;
  providerOptions?: SharedV2ProviderOptions;
}
```

##### 推理部分

用于助手消息中，捕获模型的推理过程：

```ts
interface LanguageModelV3ReasoningPart {
  type: 'reasoning';
  text: string;
  providerOptions?: SharedV2ProviderOptions;
}
```

##### 文件部分

通过包含文件在提示中，实现多模态输入：

```ts
interface LanguageModelV3FilePart {
  type: 'file';
  filename?: string;
  data: LanguageModelV3DataContent;
  mediaType: string;
  providerOptions?: SharedV2ProviderOptions;
}
```

`data` 字段提供了灵活性：

- **Uint8Array**：直接二进制数据
- **string**：Base64 编码数据
- **URL**：对外部内容的引用（如果提供商通过 `supportedUrls` 支持）

##### 工具调用部分

表示助手进行的工具调用：

```ts
interface LanguageModelV3ToolCallPart {
  type: 'tool-call';
  toolCallId: string;
  toolName: string;
  args: unknown;
  providerOptions?: SharedV2ProviderOptions;
}
```

##### 工具结果部分

包含已执行工具调用的结果：

```ts
interface LanguageModelV3ToolResultPart {
  type: 'tool-result';
  toolCallId: string;
  toolName: string;
  result: unknown;
  isError?: boolean;
  content?: Array<{
    type: 'text' | 'image';
    text?: string;
    data?: string; // base64 编码的图像数据
    mediaType?: string;
  }>;
  providerOptions?: SharedV2ProviderOptions;
}
```

可选的 `content` 字段支持丰富的工具结果，包括图像，提供了比基本 `result` 字段更多的灵活性。

### 流式传输

#### 流部分

流式传输系统为不同阶段使用类型化事件：

1.  **流生命周期事件**：

   - `stream-start`：初始事件，包含关于不支持功能的任何警告
   - `response-metadata`：模型信息和响应头
   - `finish`：最终事件，包含使用统计信息和完成原因
   - `error`：可能在任一点发生的错误事件

2.  **内容事件**：
   - 所有内容类型（`text`、`file`、`reasoning`、`source`、`tool-call`）直接流式传输
   - `tool-call-delta`：工具调用参数的增量更新
   - `reasoning-part-finish`：推理部分完成的显式标记

流序列示例：

```ts
{ type: 'stream-start', warnings: [] }
{ type: 'text', text: 'Hello' }
{ type: 'text', text: ' world' }
{ type: 'tool-call', toolCallId: '1', toolName: 'search', args: {...} }
{ type: 'response-metadata', modelId: 'gpt-4.1', ... }
{ type: 'finish', usage: { inputTokens: 10, outputTokens: 20 }, finishReason: 'stop' }
```

#### 使用量跟踪

增强的使用量信息：

```ts
type LanguageModelV3Usage = {
  inputTokens: number | undefined;
  outputTokens: number | undefined;
  totalTokens: number | undefined;
  reasoningTokens?: number | undefined;
  cachedInputTokens?: number | undefined;
};
```

### 工具

V3 规范支持两种类型的工具：

#### 函数工具

具有 JSON Schema 验证的标准用户定义函数：

```ts
type LanguageModelV3FunctionTool = {
  type: 'function';
  name: string;
  description?: string;
  parameters: JSONSchema7; // 完整的 JSON Schema 支持
};
```

#### 提供商定义的客户端工具

作为工具公开的本机提供商功能：

```ts
export type LanguageModelV3ProviderClientDefinedTool = {
  type: 'provider-defined-client';
  id: string; // 例如 'anthropic.computer-use'
  name: string; // 人类可读的名称
  args: Record<string, unknown>;
};
```

工具选择可以通过以下方式控制：

```ts
toolChoice: 'auto' | 'none' | 'required' | { type: 'tool', toolName: string };
```

### 原生 URL 支持

提供商可以声明他们可以直接访问的 URL：

```ts
supportedUrls: {
  'image/*': [/^https:\/\/cdn\.example\.com\/.*/],
  'application/pdf': [/^https:\/\/docs\.example\.com\/.*/],
  'audio/*': [/^https:\/\/media\.example\.com\/.*/]
}
```

AI SDK 在下载任何基于 URL 的内容之前检查这些模式。

### 提供商选项

规范包含一个灵活的系统，用于提供商特定功能，而不会破坏标准接口：

```ts
providerOptions: {
  anthropic: {
    cacheControl: true,
    maxTokens: 4096
  },
  openai: {
    parallelToolCalls: false,
    responseFormat: { type: 'json_object' }
  }
}
```

提供商选项可以在多个级别指定：

- **调用级别**：在 `LanguageModelV3CallOptions` 中
- **消息级别**：在单个消息上
- **部分级别**：在特定内容部分（文本、文件等）上

这种分层方法允许细粒度控制，同时保持兼容性。

### 错误处理

V2 规范强调健壮的错误处理：

1.  **流式错误**：可以通过 `{ type: 'error', error: unknown }` 在任何点发出
2.  **警告**：在 `stream-start` 和响应对象中报告的非致命问题
3.  **完成原因**：明确指示生成停止的原因：
   - `'stop'`：自然完成
   - `'length'`：达到最大 token 数
   - `'content-filter'`：安全过滤
   - `'tool-calls'`：停止以执行工具
   - `'error'`：生成失败
   - `'other'`：提供商特定原因

## 提供商实现指南

要实现自定义语言模型提供商，您需要安装所需的包：


实现自定义语言模型提供商涉及几个步骤：

- 创建入口点
- 添加语言模型实现

### 步骤 1：创建提供商入口点

首先创建一个 `provider.ts` 文件，导出一个工厂函数和一个默认实例：

```ts filename="provider.ts"
import {
  generateId,
  loadApiKey,
  withoutTrailingSlash,
} from '@ai-sdk/provider-utils';
import { ProviderV3 } from '@ai-sdk/provider';
import { CustomChatLanguageModel } from './custom-chat-language-model';

// 定义扩展 ProviderV3 的提供商接口
interface CustomProvider extends ProviderV3 {
  (modelId: string, settings?: CustomChatSettings): CustomChatLanguageModel;

  // 为不同的模型类型添加特定方法
  languageModel(
    modelId: string,
    settings?: CustomChatSettings,
  ): CustomChatLanguageModel;
}

// 提供商设置
interface CustomProviderSettings {
  /**
   * API 调用的基础 URL
   */
  baseURL?: string;

  /**
   * 身份验证的 API 密钥
   */
  apiKey?: string;

  /**
   * 请求的自定义头部
   */
  headers?: Record<string, string>;
}

// 创建提供商实例的工厂函数
function createCustom(options: CustomProviderSettings = {}): CustomProvider {
  const createChatModel = (
    modelId: string,
    settings: CustomChatSettings = {},
  ) =>
    new CustomChatLanguageModel(modelId, settings, {
      provider: 'custom',
      baseURL:
        withoutTrailingSlash(options.baseURL) ?? 'https://api.custom.ai/v1',
      headers: () => ({
        Authorization: `Bearer ${loadApiKey({
          apiKey: options.apiKey,
          environmentVariableName: 'CUSTOM_API_KEY',
          description: 'Custom Provider',
        })}`,
        ...options.headers,
      }),
      generateId: options.generateId ?? generateId,
    });

  const provider = function (modelId: string, settings?: CustomChatSettings) {
    if (new.target) {
      throw new Error(
        '模型工厂函数不能使用 new 关键字调用。',
      );
    }

    return createChatModel(modelId, settings);
  };

  provider.languageModel = createChatModel;

  return provider as CustomProvider;
}

// 导出默认提供商实例
const custom = createCustom();
```

### 步骤 2：实现语言模型

创建一个 `custom-chat-language-model.ts` 文件，实现 `LanguageModelV3`：

```ts filename="custom-chat-language-model.ts"
import { LanguageModelV3, LanguageModelV3CallOptions } from '@ai-sdk/provider';
import { postJsonToApi } from '@ai-sdk/provider-utils';

class CustomChatLanguageModel implements LanguageModelV3 {
  readonly specificationVersion = 'V3';
  readonly provider: string;
  readonly modelId: string;

  constructor(
    modelId: string,
    settings: CustomChatSettings,
    config: CustomChatConfig,
  ) {
    this.provider = config.provider;
    this.modelId = modelId;
    // 使用设置和配置进行初始化
  }

  // 将 AI SDK 提示转换为提供商格式
  private getArgs(options: LanguageModelV3CallOptions) {
    const warnings: SharedV3Warning[] = [];

    // 将消息映射到提供商格式
    const messages = this.convertToProviderMessages(options.prompt);

    // 如果提供工具则处理
    const tools = options.tools
      ? this.prepareTools(options.tools, options.toolChoice)
      : undefined;

    // 构建请求体
    const body = {
      model: this.modelId,
      messages,
      temperature: options.temperature,
      max_tokens: options.maxOutputTokens,
      stop: options.stopSequences,
      tools,
      // ... 其他参数
    };

    return { args: body, warnings };
  }

  async doGenerate(options: LanguageModelV3CallOptions) {
    const { args, warnings } = this.getArgs(options);

    // 进行 API 调用
    const response = await postJsonToApi({
      url: `${this.config.baseURL}/chat/completions`,
      headers: this.config.headers(),
      body: args,
      abortSignal: options.abortSignal,
    });

    // 将提供商响应转换为 AI SDK 格式
    const content: LanguageModelV3Content[] = [];

    // 提取文本内容
    if (response.choices[0].message.content) {
      content.push({
        type: 'text',
        text: response.choices[0].message.content,
      });
    }

    // 提取工具调用
    if (response.choices[0].message.tool_calls) {
      for (const toolCall of response.choices[0].message.tool_calls) {
        content.push({
          type: 'tool-call',
          toolCallType: 'function',
          toolCallId: toolCall.id,
          toolName: toolCall.function.name,
          args: JSON.stringify(toolCall.function.arguments),
        });
      }
    }

    return {
      content,
      finishReason: this.mapFinishReason(response.choices[0].finish_reason),
      usage: {
        inputTokens: response.usage?.prompt_tokens,
        outputTokens: response.usage?.completion_tokens,
        totalTokens: response.usage?.total_tokens,
      },
      request: { body: args },
      response: { body: response },
      warnings,
    };
  }

  async doStream(options: LanguageModelV3CallOptions) {
    const { args, warnings } = this.getArgs(options);

    // 创建流式响应
    const response = await fetch(`${this.config.baseURL}/chat/completions`, {
      method: 'POST',
      headers: {
        ...this.config.headers(),
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ ...args, stream: true }),
      signal: options.abortSignal,
    });

    // 将流转换为 AI SDK 格式
    const stream = response
      .body!.pipeThrough(new TextDecoderStream())
      .pipeThrough(this.createParser())
      .pipeThrough(this.createTransformer(warnings));

    return { stream, warnings };
  }

  // 原生文件处理支持的 URL 模式
  get supportedUrls() {
    return {
      'image/*': [/^https:\/\/example\.com\/images\/.*/],
    };
  }
}
```

### 步骤 3：实现消息转换

将 AI SDK 消息映射到您的提供商格式：

```ts filename="custom-chat-language-model.ts#L50-100"
private convertToProviderMessages(prompt: LanguageModelV3Prompt) {
  return prompt.map((message) => {
    switch (message.role) {
      case 'system':
        return { role: 'system', content: message.content };

      case 'user':
        return {
          role: 'user',
          content: message.content.map((part) => {
            switch (part.type) {
              case 'text':
                return { type: 'text', text: part.text };
              case 'file':
                return {
                  type: 'image_url',
                  image_url: {
                    url: this.convertFileToUrl(part.data),
                  },
                };
              default:
                throw new Error(`不支持的部件类型: ${part.type}`);
            }
          }),
        };

      case 'assistant':
        // 处理包含文本、工具调用等的助手消息
        return this.convertAssistantMessage(message);

      case 'tool':
        // 处理工具结果
        return this.convertToolMessage(message);

      default:
        throw new Error(`不支持的消息角色: ${message.role}`);
    }
  });
}
```

### 步骤 4：实现流式传输

创建一个流式转换器，将提供商数据块转换为 AI SDK 流部分：

```ts filename="custom-chat-language-model.ts#L150-200"
private createTransformer(warnings: SharedV3Warning[]) {
  let isFirstChunk = true;

  return new TransformStream<ParsedChunk, LanguageModelV3StreamPart>({
    async transform(chunk, controller) {
      // 使用第一个数据块发送警告
      if (isFirstChunk) {
        controller.enqueue({ type: 'stream-start', warnings });
        isFirstChunk = false;
      }

      // 处理不同的数据块类型
      if (chunk.choices?.[0]?.delta?.content) {
        controller.enqueue({
          type: 'text',
          text: chunk.choices[0].delta.content,
        });
      }

      if (chunk.choices?.[0]?.delta?.tool_calls) {
        for (const toolCall of chunk.choices[0].delta.tool_calls) {
          controller.enqueue({
            type: 'tool-call-delta',
            toolCallType: 'function',
            toolCallId: toolCall.id,
            toolName: toolCall.function.name,
            argsTextDelta: toolCall.function.arguments,
          });
        }
      }

      // 处理完成原因
      if (chunk.choices?.[0]?.finish_reason) {
        controller.enqueue({
          type: 'finish',
          finishReason: this.mapFinishReason(chunk.choices[0].finish_reason),
          usage: {
            inputTokens: chunk.usage?.prompt_tokens,
            outputTokens: chunk.usage?.completion_tokens,
            totalTokens: chunk.usage?.total_tokens,
          },
        });
      }
    },
  });
}
```

### 步骤 5：处理错误

使用标准化的 AI SDK 错误实现一致的错误处理：

```ts filename="custom-chat-language-model.ts#L250-280"
import {
  APICallError,
  InvalidResponseDataError,
  TooManyRequestsError,
} from '@ai-sdk/provider';

private handleError(error: unknown): never {
  if (error instanceof Response) {
    const status = error.status;

    if (status === 429) {
      throw new TooManyRequestsError({
        cause: error,
        retryAfter: this.getRetryAfter(error),
      });
    }

    throw new APICallError({
      statusCode: status,
      statusText: error.statusText,
      cause: error,
      isRetryable: status >= 500 && status < 600,
    });
  }

  throw error;
}
```
