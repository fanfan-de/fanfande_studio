  
这是一个非常成熟且工程化程度极高的 **Electron + React + TypeScript Monorepo** 架构。仿照这个结构开发你自己的 AI Agent 前端项目，是一个非常好的起点，特别是如果你打算开发一个功能复杂的**桌面端应用**。 为了帮助你从零开始搭建，我将把这个庞大的结构拆解为**核心骨架**，并提供一个**分步实施计划**。 ### 1. 技术选型 (复刻 Cherry Studio) 基于该目录结构，你的核心技术栈应为： * **包管理器**: `pnpm` (用于管理 Monorepo 工作区) * **构建工具**: `electron-vite` (最现代化的 Electron 开发构建工具) * **框架**: `Electron` (外壳), `React` (UI), `Express/Koa` (可选，参考它的 apiServer) * **语言**: `TypeScript` (全栈类型安全) * **测试**: `Vitest` (单元测试), `Playwright` (E2E测试) * **样式**: `TailwindCSS` (推荐) 或 `SCSS` * **AI 交互**: `Vercel AI SDK` 或直接调用 OpenAI/Anthropic 接口 --- ### 2. 推荐的项目结构 (精简版) Cherry Studio 的结构包含了大量生产环境的细节（如多语言、自动更新、补丁）。在初期，建议采用以下精简后的结构： ```text my-ai-agent/ ├── package.json # 根依赖 (eslint, typescript, electron-builder) ├── pnpm-workspace.yaml # 定义工作区 (packages/*) ├── electron-builder.yml # 打包配置 ├── electron.vite.config.ts # 构建配置 ├── .npmrc # pnpm 配置 ├── src/ # --- 核心应用源码 --- │ ├── main/ # Electron 主进程 (Node.js 环境) │ │ ├── index.ts # 入口 │ │ ├── services/ # 本地服务 (如本地向量库、文件操作) │ │ └── api/ # IPC 处理或本地 API Server │ ├── preload/ # 预加载脚本 (连接 Main 和 Renderer) │ │ └── index.ts │ └── renderer/ # 渲染进程 (React 前端) │ ├── index.html │ └── src/ │ ├── components/ # UI 组件 │ ├── hooks/ # React Hooks (useChat 等) │ ├── store/ # 状态管理 (Zustand/Redux) │ └── pages/ # 路由页面 ├── packages/ # --- 共享模块 (Monorepo) --- │ ├── ai-core/ # 核心 AI 逻辑 (独立于 UI) │ │ ├── llm/ # 模型调用封装 │ │ ├── prompts/ # 提示词管理 │ │ └── tools/ # Agent 工具函数 (搜索、代码执行) │ └── shared/ # 前后端共享的类型定义、工具函数 │ └── types/ └── resources/ # 静态资源 (图标、默认配置) ``` --- ### 3. 分步搭建指南 #### 第一步：初始化 Monorepo 环境 1. **创建文件夹并初始化**： ```bash mkdir my-ai-agent && cd my-ai-agent pnpm init ``` 2. **创建 `pnpm-workspace.yaml`**： ```yaml packages: - 'packages/*' - '.' ``` 3. **安装核心依赖**： ```bash pnpm add -D electron electron-vite vite typescript ``` #### 第二步：搭建 `packages` (核心逻辑分离) Cherry Studio 的精髓在于 `packages/aiCore`。你应该把 AI 逻辑从 UI 中剥离出来。 1. **创建 `packages/shared`**： * 存放 `Message`, `AgentConfig` 等 TypeScript 接口定义，供 Main 和 Renderer 共同引用。 2. **创建 `packages/ai-core`**： * 这里封装 LLM 的调用（OpenAI, Ollama 等）。 * 实现 "Agent" 逻辑（例如：思考链、工具调用 Loop）。 #### 第三步：配置 Electron (Main & Renderer) 使用 `electron-vite` 可以极大地简化配置。 1. **配置 `electron.vite.config.ts`**： ```typescript import { defineConfig, externalizeDepsPlugin } from 'electron-vite' import react from '@vitejs/plugin-react' export default defineConfig({ main: { plugins: [externalizeDepsPlugin()] }, preload: { plugins: [externalizeDepsPlugin()] }, renderer: { plugins: [react()] } }) ``` 2. **编写主进程 (`src/main/index.ts`)**： * 初始化窗口。 * **关键点**：设置 IPC (Inter-Process Communication) 监听。因为 AI Agent 可能需要读写本地文件、调用 Python 脚本或执行系统命令，这些必须在主进程完成。 #### 第四步：实现 AI Agent 核心功能 (参考 Cherry Studio) Cherry Studio 有一个 `apiServer` 目录，这暗示它可能在本地起了一个 Node 服务来处理请求，或者使用了类似 LangChain 的逻辑。 **你需要实现的三个核心模块：** 1. **会话管理 (Session Manager)**： * 在 `src/renderer/store` 中管理当前对话列表。 * 将历史记录持久化存储（推荐使用 `SQLite` 或 `PouchDB` 存放在本地 `resources/database`）。 2. **模型适配层 (Provider Adapter)**： * 模仿 `packages/ai-sdk-provider`。 * 编写一个通用的接口 `generateStream(messages, config)`，然后为 OpenAI, Claude, Ollama 分别实现适配器。 3. **插件/工具系统 (Plugin System)**： * Agent 的核心是使用工具。 * 在 `src/main/services/tools` 中定义工具（例如：`readFile`, `googleSearch`）。 * 通过 IPC 让前端触发这些工具。 --- ### 4. 关键代码片段示例 **1. 定义通信协议 (`packages/shared/ipc.ts`)** ```typescript export const IPC_CHANNELS = { SEND_MESSAGE: 'ai:send-message', ON_TOKEN: 'ai:on-token', // 流式输出 TOOL_EXECUTION: 'agent:execute-tool' }; ``` **2. 主进程处理 AI 请求 (`src/main/apiServer/routes/chat.ts`)** ```typescript import { ipcMain } from 'electron'; import { streamText } from 'ai'; // 假设使用 Vercel AI SDK import { openai } from '@ai-sdk/openai'; ipcMain.on('ai:send-message', async (event, { messages }) => { const result = await streamText({ model: openai('gpt-4o'), messages, }); for await (const textPart of result.textStream) { event.sender.send('ai:on-token', textPart); } }); ``` ### 5. 进阶建议 (针对 Agent 项目) 1. **流式传输 (Streaming)**：必须实现打字机效果。Electron 中通过 `WebContents.send` 持续发送 Token 到前端。 2. **本地向量库**：如果你要做知识库（RAG），可以引入 `chromadb` 的 JS 客户端或者使用 `use-llm` 等库在本地运行简单的 embeddings。 3. **安全策略**：注意 `electron/main/index.ts` 中的 CSP (Content Security Policy)，Agent 可能会请求各种外部 API，需要配置好白名单。 4. **配置隔离**：参考 `config/` 目录，将用户的 API Key 存储在操作系统的安全存储中（使用 `keytar` 或 Electron 的 `safeStorage`），不要明文存文件。 ### 总结 模仿 Cherry Studio 的结构，本质上是在搭建一个 **"可扩展的本地 AI 操作系统"**。 * **起步**：先跑通 Electron + React + Vite。 * **核心**：打通 UI 输入 -> IPC -> Main Process (调用 AI SDK) -> IPC -> UI 流式输出。 * **扩展**：再考虑 Monorepo 拆分 packages 和添加插件系统。
