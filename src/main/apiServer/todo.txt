3. API 服务器模块 (src/main/apiServer/)
功能概述： 内置的 Express 服务器，提供 RESTful API 接口供外部系统集成，支持 AI 对话、模型管理、文件操作等功能。

核心功能：

AI 对话接口：提供标准化聊天补全 API，支持流式响应
模型管理：列出可用 AI 模型和配置信息
消息历史：管理对话会话和消息存储
MCP 集成：支持 Model Context Protocol 服务器管理
配置管理：动态调整服务器设置
技术栈：

Express.js：轻量级 Web 框架
Swagger：自动生成 API 文档
中间件系统：认证、错误处理、请求验证等
流式响应：支持 Server-Sent Events (SSE)
关键文件：

src/main/apiServer/server.ts - HTTP 服务器实现
src/main/apiServer/app.ts - Express 应用配置
src/main/apiServer/routes/ - 路由定义
chat.ts - 聊天补全接口
models.ts - 模型管理接口
messages.ts - 消息历史接口
mcp.ts - MCP 协议接口
src/main/apiServer/services/ - 业务逻辑层
src/main/apiServer/config.ts - 服务器配置
架构特点：

模块化路由：每个功能模块独立路由文件
服务层分离：业务逻辑与路由层分离，提高可测试性
配置驱动：支持动态重新加载配置
健康检查：内置服务器状态监控
安全特性：

请求超时控制（默认 5 分钟）
请求头大小限制
连接保活管理
错误信息脱敏
4. 配置管理模块 (src/main/services/ConfigManager.ts)
功能概述： 集中化的应用配置管理系统，负责存储和同步用户设置、应用偏好和系统配置。使用 electron-store 提供持久化存储。

核心配置项：

界面设置：语言、主题、缩放比例
系统行为：启动到托盘、关闭到托盘、自动更新
AI 功能：选择助手启用、触发模式、过滤设置
开发者选项：开发者模式、硬件加速禁用
客户端标识：唯一客户端 ID 生成
技术实现：

存储引擎：基于 electron-store 的 JSON 文件存储
类型安全：强类型配置键和值定义
变更通知：观察者模式支持配置变更监听
默认值管理：智能默认值回退机制
关键接口：

export class ConfigManager {
  getLanguage(): LanguageVarious
  setLanguage(lang: LanguageVarious): void
  getTheme(): ThemeMode
  setTheme(theme: ThemeMode): void
  getLaunchToTray(): boolean
  // ... 其他配置方法
}
特殊处理：

平台适配：不同操作系统的默认配置差异
配置迁移：支持配置版本升级和数据迁移
订阅机制：组件可以订阅特定配置项的变更
