# Security Policy

## Reporting A Vulnerability

Please do not open a public GitHub issue for suspected vulnerabilities.

Report security issues by contacting the project maintainers privately through the email address listed on the GitHub organization or repository owner profile. Include:

- A clear description of the issue.
- Steps to reproduce or a proof of concept.
- Affected versions, platforms, or configurations.
- Any known mitigations.

## Supported Versions

Security fixes target the current `master` branch and the latest published release unless a maintainer states otherwise.

## Local Data Notice

Anybox works with local project folders, terminal sessions, model/provider configuration, and agent traces. Do not commit local databases, logs, screenshots, exported artifacts, or personal workspace notes. The repository `.gitignore` includes common patterns for these files.
