# 会话线程拖拽弹出为独立系统窗口

## Summary
在当前单 `BrowserWindow` 的 Electron 架构上，新增“会话级独立窗口”能力：用户从主窗口左侧会话项发起拖拽，拖出后创建一个真正的系统新窗口，只承载该会话的 `ThreadView + Composer + RightSidebar`。  
第一版采用“会话所有权转移”模型：一个会话同一时间只由一个窗口负责交互；主窗口侧栏保留该会话入口和“已拆出”状态，点击时只聚焦独立窗口，不再在主内容区重复渲染。

## Key Changes
### Main process
- 为窗口增加显式上下文描述：`main` 或 `detached-session(sessionID)`。
- 引入 detached session 注册表，保证“一个 session 只对应一个独立窗口”；重复拖拽或点击已拆出会话时直接聚焦现有窗口。
- 扩展窗口创建函数，支持创建 session window，并把窗口标题设为会话标题。
- 在独立窗口关闭时执行 redock：清理注册表并向主窗口广播 detached 状态变更。

### Preload / IPC / public interfaces
新增桌面桥接接口，保持 agent 服务端 API 不变：
- `getWindowContext(): Promise<{ kind: "main" | "detached-session"; sessionID?: string }>`
- `popoutSession(input: { sessionID: string; screenX: number; screenY: number }): Promise<{ focusedExisting: boolean }>`
- `focusDetachedSession(input: { sessionID: string }): Promise<boolean>`
- `listDetachedSessions(): Promise<string[]>`
- `onDetachedSessionsChange(listener: (sessionIDs: string[]) => void): () => void`

新增或扩展的 renderer 类型：
- `WindowContext`
- `DetachedSessionState`
- `SessionSummary.windowState: "docked" | "detached"`

### Renderer architecture
- 把当前 `App` 拆成两个壳：
  - `MainWorkspaceShell`：保留现有左侧栏、主画布、终端、设置页。
  - `DetachedSessionShell`：仅渲染 `Titlebar + ThreadView + Composer + RightSidebar`。
- 把会话运行态从当前大一统的 `useAgentWorkspace` 中抽离，形成可复用的“按 `sessionID` 加载 history/diff/permission/send/composer”的会话级 hook；主窗口和独立窗口都复用这套会话逻辑。
- 主窗口增加 detached session 订阅：
  - 被拆出的会话在侧栏显示独立标记。
  - 若当前 active session 被拆出，主窗口自动切到最近一个未拆出的会话；没有则显示空态。
  - 点击已拆出会话时调用 `focusDetachedSession`，不在主画布重复打开。
- 独立窗口按 `sessionID` 直接加载该会话的历史、diff、权限请求和发送能力；不包含左侧工作区、terminal、settings。

### Drag interaction
- 拖拽入口放在左侧会话项，使用专门的拖拽手柄，避免破坏现有点击选中行为。
- 使用自定义 pointer 事件，不使用原生 HTML5 drag-and-drop。
- 行为规则：
  - `pointerdown` 进入候选状态；
  - 超过拖拽阈值后进入 detaching 状态；
  - `pointerup` 时以释放点 `screenX/screenY` 调用 `popoutSession`；
  - 主进程在释放点附近创建新窗口，并裁剪到目标显示器工作区；
  - 若该会话已拆出，则只聚焦现有窗口。
- 会话删除时，如果该会话在独立窗口中，先关闭该窗口，再清理 UI/注册表。

## Tests
- Main 进程单测：
  - 同一 session 不会创建两个独立窗口。
  - 关闭 detached window 会正确 redock 并广播状态。
  - `focusDetachedSession` 在 session 已拆出和未拆出时返回正确结果。
- Renderer 单测：
  - 拖拽会话项超过阈值会调用 `popoutSession`。
  - 主窗口收到 detached 状态后会给会话项打标，并阻止主画布重复渲染该会话。
  - 点击已拆出会话会调用 `focusDetachedSession`。
  - detached window 模式只渲染 `ThreadView + Composer + RightSidebar`，不渲染左侧栏和 terminal。
  - detached window 中发送消息、查看 diff、处理权限请求仍走原有 session 级 IPC。
- 回归测试：
  - 普通未拆出会话的切换、发送、删除不受影响。
  - 终端面板、设置页、窗口最大化/最小化逻辑保持原行为。

## Assumptions
- 第一版只支持“会话线程”拆出，不支持任意面板自由拆出。
- 独立窗口默认内容固定为 `线程 + 输入框 + 变更侧栏`。
- 会话拖出后采用“转移归属”，主副窗不同时可编辑同一会话。
- 关闭独立窗口时会话自动回到主窗口；未发送 draft/附件状态不做跨窗口持久化。
- 不改 fanfandeagent 协议，只在 desktop shell 层增加窗口管理与桥接。
- 同步更新前端和桌面 API 文档，覆盖新的窗口上下文与 detached session 生命周期。
