# Obsidian UI Spec

适用范围：本仓库后续所有前端 / 桌面端 UI 开发，除非用户明确要求别的风格。

## 目标风格

- 默认做成 `Obsidian` 风格的桌面工具界面，不做 SaaS 官网风，不做卡片堆砌风。
- 关键词：克制、安静、高信息密度、结构清晰、弱装饰、强布局。
- 视觉重点来自 `分栏 / 层级 / 对齐 / 间距 / 轻微状态色`，不是来自大面积边框、胶囊按钮或营销式文案。

## 布局规则

- 优先使用 `sidebar / split pane / tabs / inspector / list-detail` 这类桌面应用结构。
- 面板之间主要靠 `1px 分隔线`、背景层级差和间距区分，不靠一堆 card。
- 同一层级内尽量保持平直、规整、贴边，不要出现过多悬浮块。

## 控件规则

- `button` 默认使用 `ghost / borderless / flat` 风格。
- 不要默认给顶部菜单按钮、侧栏按钮、工具栏按钮加外框。
- 只有在 `primary action` 明确需要强调时，才允许使用有底色按钮；也应保持克制。
- `icon` 能表达清楚时，直接使用 icon，不再附带文字。
- 像 `Left Sidebar Top Menu`、`Right Sidebar Top Menu` 这类稳定位置的导航，优先 `icon-only`。
- `active` 状态优先使用 `颜色变化 / 微弱底色 / 左边条 / 下划线`，不要厚重描边。
- 圆角保持小或无圆角，避免药丸按钮和过度圆润的组件。

## 文案规则

- 不写多余的解释性文字，不写“为了说明结构而存在”的注释型 UI 文案。
- 不要把设计结构名直接渲染到界面上，例如：
  - `Top Menu`
  - `Sidebar Actions`
  - `Inspector Panel`
- 能省略的 label 就省略，能靠上下文理解的就不要重复写。
- 空状态文案保持短句，直接说明当前状态，不要写大段引导。

## 视觉规则

- 优先使用低饱和中性色；若无特别要求，偏 `slate / gray / muted blue-gray`。
- 可以保留 Obsidian 式的桌面感和轻微暗色氛围，但如果当前页面已有明确主题，则延续现有主题，不强行切黑。
- 阴影、渐变、玻璃态都应极轻；没有明确需要时宁可不用。
- 图标使用简单线性 icon，避免拟物、彩色插画和装饰徽章。

## 明确禁止

- 外框明显的 toolbar button / tab button / sidebar button。
- 为了“看起来丰富”而添加的副标题、说明文字、注释标签。
- 大量 card、胶囊标签、彩色模块、营销感 banner。
- 无实际作用的图标 + 文字重复表达。
- 把界面做成“演示页”而不是“可工作的桌面工具”。

## 默认决策

如果需求不够明确，按下面顺序决策：

1. 优先更像 `Obsidian`，而不是更像通用 SaaS 后台。
2. 优先减少文字，而不是补充说明。
3. 优先去掉按钮框，而不是额外加框。
4. 优先使用 panel 和 separator，而不是 card 和大块底色。
5. 优先让 UI 看起来是“长期使用的工具”，而不是“首次展示的页面”。

## 开发前自检

- 这个文字是不是可以删掉？
- 这个按钮是不是可以只保留 icon？
- 这个按钮边框是不是可以去掉？
- 这个区域是不是应该用分栏/分隔线，而不是 card？
- 这个界面看起来像不像 Obsidian，而不是像模板站？
