## 📋 模块概述

`src/acp` 模块是 OpenCode 的 **Agent Client Protocol (ACP)** 实现，它允许 OpenCode 作为一个标准化协议服务器运行，与支持 ACP 的客户端（如 Zed、Cursor 等编辑器）进行通信。

## 🏗️ 架构设计

模块采用清晰的分层架构：

### 1. **agent.ts** - 协议核心
- 实现了 `@agentclientprotocol/sdk` 中的 `Agent` 接口
- 处理所有 ACP 协议请求：`initialize`、`authenticate`、`newSession`、`loadSession`、`prompt` 等
- 负责实时事件订阅和推送（工具调用状态、消息更新等）
- 权限请求处理和用户确认流程
- 模型选择和模式切换管理

### 2. **session.ts** - 会话管理
- `ACPSessionManager` 类管理 ACP 会话生命周期
- 将 ACP 会话映射到 OpenCode 内部会话系统
- 维护工作目录、MCP 服务器配置、模型选择等状态
- 支持会话的创建、加载、恢复和分叉

### 3. **types.ts** - 类型定义
- `ACPConfig` - ACP 服务器配置（SDK 客户端、默认模型等）
- `ACPSessionState` - ACP 会话状态数据结构

### 4. **README.md** - 详细文档
- 架构说明、协议合规性、使用指南
- 当前限制和未来增强计划

## 🔗 与其他模块的集成

### 与 OpenCode SDK 集成
- 使用 `@opencode-ai/sdk/v2` 与 OpenCode 后端通信
- 通过 SDK 调用会话、工具、配置等 API

### 与权限系统集成
- 监听 `permission.asked` 事件，向客户端请求权限确认
- 支持 "Allow once"、"Always allow"、"Reject" 三种权限选项
- 自动处理文件编辑等操作的权限验证

### 与 MCP 系统集成
- 支持客户端传递的 MCP 服务器配置
- 动态添加和管理 MCP 服务器连接

### 与工具系统集成
- 将 ACP 工具调用映射到 OpenCode 内部工具（bash、edit、read 等）
- 实时报告工具执行状态和结果

## 📡 协议功能实现

### ✅ 已完整实现的功能
1. **协议初始化** - 版本协商、能力声明、认证方法
2. **会话管理** - 新建、加载、列表、分叉、恢复会话
3. **提示处理** - 处理用户输入（文本、图片、文件资源）
4. **实时更新** - 工具调用状态、消息流、思考过程
5. **客户端能力** - 文件读写、权限请求、命令执行
6. **模型管理** - 模型选择、变体支持、模式切换

### 🔄 工作流程示例
```
1. 客户端（如Zed）通过 stdio 启动 `opencode acp` 命令
2. 发送 `initialize` 请求，协商协议版本
3. 创建新会话 `newSession`，指定工作目录和 MCP 服务器
4. 用户输入提示，客户端发送 `prompt` 请求
5. ACP Agent 调用 OpenCode SDK 处理提示
6. 实时推送工具调用、消息块、思考过程等更新
7. 返回最终结果，客户端显示给用户
```

## 🎯 模块的核心价值

### 1. **标准化接口**
- 使 OpenCode 能够与任何支持 ACP 的客户端兼容
- 遵循行业标准协议，降低集成成本

### 2. **实时交互**
- 支持流式响应，提供更好的用户体验
- 实时显示工具执行进度、思考过程

### 3. **权限管理**
- 安全的权限请求机制，用户可控制工具访问
- 支持一次性和永久授权

### 4. **会话持久化**
- 支持会话恢复和历史回放
- 保持工作上下文和配置状态

### 5. **扩展性**
- 支持 MCP 服务器动态添加
- 可扩展的模型和模式系统
