# Util Module Spec

## 概述
`util` 模块放的是跨模块复用的基础工具。这里的代码应该尽量小而稳定，只解决“通用能力”，不承载业务逻辑。

## 核心职责
- 日志、错误和上下文容器
- 文件系统与路径辅助
- 队列、锁、惰性计算等基础并发工具
- 通用字符串、 slug、which 等小工具

## 主要文件
- `context.ts`：上下文容器和作用域辅助
- `filesystem.ts`：文件系统发现、读取和路径判断
- `error.ts`：统一错误封装
- `queue.ts`：并发队列
- `lock.ts`：读写锁
- `log.ts`：结构化日志

## 约束
- 工具函数应保持无副作用或低副作用
- 不要把业务规则放进 util
- 任何跨模块共享能力都应尽量保持 API 简单
- 日志、错误、路径处理等基础设施应优先保持稳定
