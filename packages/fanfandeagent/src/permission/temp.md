1.工具列表的动态注入：根据用户权限，动态生成 Prompt。例如，根据用户角色，只提供相应的工具描述、API 签名等。
2.LLM 输出的解析与参数校验（执行前校验）（防范的是LLM的危险的输出）
	**校验大模型生成的工具调用请求是否合法**
	- **参数格式校验：** 检查大模型输出的参数是否符合工具的 API 定义（例如，参数的数据类型、必须的字段是否都存在）。
    - **语义校验：** 检查参数的含义是否合理（例如，用户 ID 是否存在，要删除的文件是否属于用户）。
    - **拒绝恶意参数：** 过滤包含敏感词、恶意代码的参数。
	- **值范围校验：** 检查参数值是否在允许的范围内（例如，数值是否在指定区间内，枚举值是否是合法的选项）。
3.**执行前/后权限校验和上下文绑定（后端拦截与控制）：**
	**确保只有授权用户才能使用工具，并且操作的数据是其有权限访问的。** 这个环节是核心的权限控制，即使大模型生成了正确的参数，后端也需要再次验证。
	- **执行前（Pre-Execution）：** 在执行工具前，根据当前用户身份和权限，对参数进行校验。例如，检查用户是否有权限删除某个文件，或者是否有权限访问某个数据库表。
	- **上下文绑定：** 在调用工具时，强制将当前用户的身份信息（例如用户 ID）传递给工具函数。
	- **执行后（Post-Execution）：** 在工具执行完成后，对结果进行处理。例如，过滤敏感信息、记录审计日志。
4.人在环节的验证
- **作用：** **在关键操作之前，增加人工确认，降低风险。** 对于高风险操作，例如转账、发送邮件，不直接信任大模型的决策。
- **实现方式：** 在执行操作前，提示用户进行确认。在某些情况下，可能需要多重身份验证。
