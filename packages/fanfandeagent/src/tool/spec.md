# Tool Module Spec

## 概述
`tool` 模块负责定义和验证所有可供模型调用的工具。它不负责业务实现本身，而是负责把工具描述成稳定、可校验、可执行的接口。

## 核心职责
- 定义工具元数据和执行上下文
- 在执行前校验输入参数
- 把模型侧的工具调用转换成可执行函数
- 提供统一的错误格式化入口

## 主要文件
- `tool.ts`：工具定义器和运行时包装
- `registry.ts`：工具注册与可用性管理
- `bash.ts`：命令类工具实现

## 核心类型
- `ToolInfo`：工具定义元数据
- `InitContext`：初始化时可用的上下文
- `Context`：工具执行时可用的 session / message 信息

## 约束
- 工具参数必须在执行前通过 Zod 校验
- 工具描述和执行函数应保持一致
- 工具层不应直接关心持久化实现
- 工具结果应尽量保持结构简单，便于写入消息历史
