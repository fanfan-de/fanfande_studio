# FanfandeAgent Backend Backlog

更新日期：2026-04-07

这份 backlog 基于当前后端代码状态整理，目标不是继续堆功能，而是先把 Agent 主链路稳定下来，再补核心能力，最后再做平台化建设。

## 当前判断

当前项目更接近“可运行的 Agent 原型后端”，还不是“可稳定扩展的生产级 Agent 后端”。

已经具备雏形的模块：
- `project / instance`
- `server / routes`
- `session / message / part` 持久化
- `tool` 基础执行
- `permission` 权限与审批基础能力

当前最明显的短板：
- 项目级配置和全局配置混用
- `provider` 与 `llm` 编排没有真正闭环
- `session processor` 生命周期不完整
- 长会话压缩、标题摘要、快照恢复没有接入主流程
- 测试不稳定，类型检查未通过

## 执行原则

- 先完成 `P0`，确保主链路稳定、可测试、可维护。
- 再完成 `P1`，补足 Agent 后端必须具备的核心运行能力。
- 最后完成 `P2`，再做平台化、运维化、长期演进相关能力。

## P0 基础稳定性

阶段目标：把“项目 -> 会话 -> 模型 -> 工具 -> 持久化 -> 响应”这条主链路做成稳定闭环。

### 1. 项目级配置隔离

- [ ] 将 `provider / model / api key / policy` 从全局配置切换为真正的项目级配置

验收标准：
- 不同项目可以配置不同模型和不同 provider
