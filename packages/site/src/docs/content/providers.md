# 模型供应商

Anybox 支持通过 Provider 和 Model 配置选择不同的大模型服务。
基本上
## 配置位置

在桌面端设置中进入模型配置区域，按供应商填写凭据并选择默认模型。一次会话也可以单独指定模型，让不同任务使用不同的推理能力和成本档位。

## 推荐配置顺序

1. 先添加一个稳定可用的 Provider。
2. 选择一个默认模型。
3. 运行一个只读问题验证连接。
4. 再为写代码、重构或长上下文任务配置更强的模型。

如果你不确定从哪里开始，先配置一个常用云端供应商和一个日常模型即可。后续可以按任务复杂度补充更多模型。

## 凭据安全

不要把 API Key 写入仓库。团队协作时，应把密钥放在本机环境、系统凭据管理器或部署环境变量中。

```powershell
# 示例：只在当前 PowerShell 会话中设置环境变量
$env:PROVIDER_API_KEY="your-api-key"
```

如果你连接的是外部 Agent 服务，请确认服务地址、网络访问范围和认证方式符合团队安全要求。

## 排障

模型不可用时，优先检查：

- API Key 是否有效。
- 供应商账户是否有额度。
- 当前网络是否能访问供应商 API。
- 选择的模型 ID 是否仍然可用。
- 会话是否连接到了预期的本地或远程 Agent 服务。
