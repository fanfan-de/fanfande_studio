# Skills

Skills 用来把重复的工作方法沉淀成可复用指令。它适合保存项目规范、发布流程、测试约定、工具调用方式和团队偏好。

## 适合写成 Skill 的内容

- 项目构建、测试、发布流程。
- 代码风格和架构边界。
- 常见任务的检查清单。
- 需要固定顺序执行的工作流。
- 特定工具或服务的使用规则。

不要把临时聊天记录、密钥、私人账号或一次性任务写入 Skill。

## 基本结构

一个 Skill 通常由 `SKILL.md` 描述。建议包含清晰的触发条件和可执行步骤。

```md
---
name: release-check
description: 发布前检查构建、测试和产物。
---

# Release Check

当需要发布版本时使用。

1. 运行类型检查。
2. 运行测试。
3. 构建产物。
4. 检查发布说明和版本号。
```

## 使用建议

保持 Skill 短而具体。一个 Skill 只解决一类问题，比把所有团队规范写成一份超长说明更容易维护。

## 管理方式

你可以为个人常用流程维护全局 Skills，也可以为项目维护项目级 Skills。项目级 Skills 更适合放置和当前仓库强相关的构建、测试和发布规则。
