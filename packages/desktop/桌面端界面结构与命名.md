结构补充说明：

1. `Window Shell` 当前只有浮动窗口控制，没有独立 `Titlebar` 容器，也没有 `titlebar*` 命名锚点。
2. `Activity Rail` 只存在左侧，没有右侧对称 rail。
3. `Canvas` 主体有两个互斥内容态：`Create Session Canvas`，或 `Thread Shell + Composer`。
4. `Session Canvas Top Menu` 只要当前不是 create-session tab 就会渲染；`activeSession` 为空时显示 `No session selected`。
5. `Canvas Region Top Menu` 始终承载右侧 Inspector 的切换按钮；`Right Sidebar Top Menu` 只负责右侧视图按钮，不负责折叠/恢复。
6. 左侧折叠按钮位置由壳层状态决定：左侧窄轨显示时固定在 `Activity Rail`；窄轨隐藏且左栏展开时在 `Left Sidebar Top Menu`；窄轨隐藏且左栏折叠时在 `Canvas Region Top Menu` 左侧。
7. `Composer` 当前只有附件选择、模型菜单和发送按钮，没有 `Agent Mode Menu`。
8. 常规 assistant turn 使用分组化 `Assistant Sections`；`assistant-trace-list` 只用于“未选中 session”空态。
9. `Settings Page Overlay` 是 `App Shell` 的覆盖层 sibling，不是 `Canvas` 的子区域。

## 3. 术语约束

1. 左侧导航统一称“文件夹工作区”“workspace”“session”，不要混用 `project tree`、`folder tree` 指代不同层次。
2. 当前不存在独立 `Titlebar`；窗口顶部按钮统一称 `Window Controls Floating` 或“浮动窗口控制”。
3. 右侧栏统一称 `Right Sidebar` 或 `Inspector`，不要写成 `Canvas Tool Panel`。
4. 设置页一级分组统一使用 `Provider`、`Models`、`Appearance`，不要沿用旧的 `Model Services`、`Default Models`。
5. 左侧窄轨统一称 `Activity Rail` 或“左侧窄轨”，不要和左侧栏混称。
6. `Create Session Canvas` 与常规 session canvas 要分开命名；前者不是 thread 空态，也不是 composer 空态。
7. `Composer` 当前只有模型菜单，不要再写成含 `Agent Mode Menu` 的结构。

## 4. 维护规则

1. 新增稳定界面区域、ARIA label 或测试锚点时，先更新本文档，再改代码。
2. 编写测试时，优先使用本文档中的类名、ARIA label 或稳定文案定位。
3. 如果窗口控制、Canvas Top Stack、Create Session Canvas、Terminal Panel、左右侧栏切换关系或 Settings Page 结构职责变化，必须同步更新本文档和 `AI_AGENT_FRONTEND_SPEC.md`。
