结构补充说明：

1. `Window Shell` 当前只有浮动窗口控制，没有独立 `Titlebar` 容器，也没有 `titlebar*` 命名锚点。
2. 左侧栏当前不是单视图，`Left Sidebar Top Menu` 下有 `Workspace` 与 `Skills` 两个固定入口。
3. `Canvas` 当前有三种互斥主内容态：
   - `Skills` 视图下为 `Global Skills Canvas`
   - `create-session` tab 下为 `Create Session Canvas + Composer`
   - 常规 session 下为 `Thread Shell + Composer`
4. `CanvasRegionUtilityMenu` 与 `CanvasRegionTopMenu` 复用同一组 `canvas-region-top-menu*` 类名，只是中段内容从 session tabs 改成静态 utility label。
5. `Session Canvas Top Menu` 只在 `leftSidebarView !== "skills"` 且当前不是 create-session tab 时渲染；`activeSession` 为空时标题显示 `No session selected`。
6. `Session Canvas Top Menu` 当前承载的是 `Project MCP Menu`、`Project Skills Menu` 与 `Git Quick Menu`；相关选择器已经从 `Composer` 移出。
7. `Composer` 当前只有附件、模型与发送动作，没有 `Agent Mode Menu`，也没有 project skills / MCP 选择入口。
8. `Create Session Canvas` 当前只有 `Session project` 下拉框，没有独立 `Session title` 输入；首条任务输入仍然来自 `Composer`。
9. 当前支持多个 `create-session` tab；它们与常规 session tab 一起共存于 `Canvas Region Top Menu`。
10. 左侧折叠按钮位置由壳层状态决定：左侧窄轨显示时固定在 `Activity Rail`；窄轨隐藏且左栏展开时在 `Left Sidebar Top Menu`；窄轨隐藏且左栏折叠时在 `Canvas Region Top Menu` 左侧。
11. 右侧没有对称 `Activity Rail`；Inspector 的切换按钮始终在当前活跃表面上，在展开时位于右侧栏 header，折叠后回到 `Canvas Region Top Menu`。
12. `Settings Page Overlay` 是 `App Shell` 的覆盖层 sibling，不是 `Canvas` 的子区域。
13. `Settings Page` 当前一级分组是 `Provider`、`Models`、`MCP`、`Appearance` 四项；`MCP` 是新增的稳定一级页面。

## 3. 术语约束

1. 左侧导航中，`workspace` 指文件夹工作区，`session` 指工作区下的会话；不要再用 `project tree`、`folder tree` 混称。
2. 左侧第二个稳定视图统一称 `Skills` 或 `Global Skills`，不要把它写成 `Workspace Tools`、`Skill Explorer` 等别名。
3. 当前不存在独立 `Titlebar`；窗口顶部按钮统一称 `Window Controls Floating` 或“浮动窗口控制”。
4. 右侧栏统一称 `Right Sidebar` 或 `Inspector`，不要写成 `Canvas Tool Panel`。
5. 左侧窄轨统一称 `Activity Rail` 或“左侧窄轨”，不要和左侧栏混称。
6. `Canvas Region Top Menu` 与 `Canvas Region Utility Menu` 要分开表述；前者承载 tabs，后者承载 utility label。
7. `Create Session Canvas` 与常规 session canvas、`Global Skills Canvas` 都是不同内容态，不要混称为空态 thread。
8. `Composer` 当前只承担任务输入，不要再写成包含 `Agent Mode Menu`、`Project Skills Menu` 或 `Project MCP Menu` 的结构。
9. 项目级 `Skills` / `MCP` 选择器统一归属 `Session Canvas Top Menu`，不要写到 `Composer` 或右侧栏里。
10. 设置页一级分组统一使用 `Provider`、`Models`、`MCP`、`Appearance`，不要沿用旧的 `Model Services`、`Default Models` 等命名。
11. `MCP` 页负责全局 server 定义，project 只提供诊断上下文；不要把这一页写成 project 内局部面板。

## 4. 维护规则

1. 新增稳定界面区域、ARIA label、类名锚点或测试依赖时，先同步更新本文档，再改代码。
2. 编写测试时，优先使用本文档中的类名、ARIA label 或稳定文案定位。
3. 如果左侧 `Workspace/Skills` 双视图、`Canvas` 三种内容态、`Session Canvas Top Menu` 职责、`MCP` 设置页、`Create Session` 流程或左右侧栏切换关系变化，必须同步更新本文档。
