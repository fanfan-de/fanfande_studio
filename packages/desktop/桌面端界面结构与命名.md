结构补充说明：

1. `workspace` 模式已经不是单一 `Canvas Region Top Menu + Thread` 结构；当前主模式是递归 `Workbench`，每个 pane 自带一套 `Pane Tab Bar + Session Canvas Top Menu + 内容区 + Composer Stack`。
2. `CanvasRegionTopMenu` 组件当前没有挂到主壳层；workspace 顶部 tabs 的稳定称呼应为 `Pane Tab Bar`。`Skills` 模式仍然通过 `CanvasRegionUtilityMenu` 复用 `canvas-region-top-menu*` 类名。
3. `Canvas` 当前只有两种主模式：
   - `Skills` 模式: `Canvas Region Utility Menu + Global Skills Canvas`
   - `Workbench` 模式: `Workbench Panes + Terminal Area`
4. `Workbench` 支持递归水平/垂直分割、pane resizer、tab 拖拽换位、tab 拖拽拆分、tab 合并预览。
5. 左侧恢复按钮不再是全局唯一入口；它只出现在“第一个 pane 的 `Pane Tab Bar`”或 `Skills` 模式的 `Canvas Region Utility Menu` 左侧。
6. 右侧栏切换按钮也不再固定在单一全局顶部；workspace 模式下只挂在“顶行最右侧 pane 的 `Pane Tab Bar`”右侧，skills 模式则挂在 `Canvas Region Utility Menu` 右侧。
7. `Session Canvas Top Menu` 现在承载四类入口：
   - `External Editor Menu`（有目录且宿主 API 可用时显示）
   - `Project MCP Menu`
   - `Project Skills Menu`
   - `Git Quick Menu`
8. `Project MCP Menu`、`Project Skills Menu`、`Git Quick Menu` 在 side-chat session 内不显示；side-chat 只保留必要的状态文案，`Editor` 按钮仍可能显示。
9. `Composer` 现在除了附件条，还稳定承载 `Comment Reference Strip`；这些引用来自 `Preview` / `Files` 面板的批注导入。
10. `Composer Utility Bar` 当前稳定子项是：
    - `Context Pressure Indicator`
    - `Permission Mode Toggle`
    - `Git Branch Switcher`
11. `Permission Mode Toggle` 是新增稳定入口，控制当前 composer 使用默认审批策略还是 full access；side-chat 中不显示。
12. `Right Sidebar` 当前不再只有 `Changes`，而是四视图：
    - `Changes`
    - `Runtime`
    - `Preview`
    - `Files`
13. `Preview` 面板支持 `Browse / Comment` 两种交互模式；Electron webview 初始化失败时会回退到 iframe 模式，并只在回退模式下显示坐标点击评论覆盖层。
14. `Files` 面板支持文件名搜索、文本文件阅读、单行/区间批注、批注确认与插入草稿。
15. 左侧 `Session Tree` 行尾动作现在是“归档”而不是“永久删除”；归档后的会话统一在 `Settings > Archived Sessions` 中恢复或彻底删除。
16. `Settings` 顶级分组已经扩展为六项：
    - `Provider`
    - `Models`
    - `MCP`
    - `Appearance`
    - `Developer Mode`
    - `Archived Sessions`
17. `Appearance` 只负责主题与壳层可见性；开发期调试色、调试线颜色、trace 可见性已经迁移到 `Developer Mode`。
18. workspace 模式下终端折叠按钮会优先挂到 `Activity Rail Bottom Slot`；只有左 rail 不可用时才回落到 `Canvas Terminal Toggle Anchor`。

## 2. 开发期区域调试色规范

本节只用于结构沟通、布局排查与开发期对齐，不代表最终视觉稿。

调试色使用规则：

1. 调试背景色只服务于结构沟通，不代表品牌色，也不参与最终视觉验收。
2. 同一个稳定区域在不同页面态下尽量保持同一背景色，不要频繁换色。
3. 先给大区域上色，再给必要的小区域补色；不要把每个按钮都染成不同颜色。
4. 子区域如需继续拆分，优先沿用父区域同色系的浅深变化，不额外发明新主色。
5. 互斥内容态可以各自有独立背景色，但文档里必须明确“同一时刻不会同时出现”。
6. `Pane Tab Bar` 与 `Canvas Region Utility Menu` 当前共用同一套浅黄色调；前者对应 workspace workbench tabs，后者对应 skills 顶部 utility bar。
7. 当前调试色与调试线颜色都应从 `Settings > Developer Mode` 控制：
   - `Show debug region colors`
   - `Show line debug colors`
8. 正式视觉收敛后可以逐步移除调试色，但区域名称与本文档映射需要保留，作为沟通基线。

5. `Show line debug colors` 只用来区分顶部残余细线来源，不改变区域命名；重点是让 `Pane Tab Bar` 的下边线与其他壳层边线可一眼区分。

## 4. 术语约束

1. `workspace` 指文件夹工作区，`session` 指工作区下的主会话；不要再用 `project tree`、`folder tree` 混称。
2. workspace 模式顶部 tabs 统一称 `Pane Tab Bar`，不要再把它叫做 `Canvas Region Top Menu`。
3. `Canvas Region Utility Menu` 只用于 `Skills` 模式顶部工具条；虽然代码锚点复用 `canvas-region-top-menu*`，沟通时仍应明确说“技能顶部工具栏”。
4. 当前不存在全局单一 `Canvas Region Top Menu + Session Canvas Top Menu + Thread` 的旧壳层；主结构是多 pane `Workbench`。
5. 右侧栏统一称 `Right Sidebar` 或 `Inspector`；当前稳定四视图是 `Changes`、`Runtime`、`Preview`、`Files`。
6. `Preview` 指内置网页预览与评论，不要混称为浏览器、webview 面板或 dev server iframe。
7. `Files` 指工作区文件搜索 / 阅读 / 批注面板，不要混称为 diff viewer。
8. `Composer` 只负责草稿、评论引用、附件和发送动作；项目级 `MCP` / `Skills` / `Git Quick Menu` 不属于 `Composer`。
9. `Composer Utility Bar` 统一承载 `Context Pressure Indicator`、`Permission Mode Toggle` 与 `Git Branch Switcher`；不要把这些归到 `Composer Toolbar` 或顶部工具条。
10. `Permission Mode Toggle` 是 composer 级权限策略切换，不要混称为全局设置项、右侧栏开关或 session 状态徽标。
11. `Session Canvas Top Menu` 当前承载 `External Editor Menu`、`Project MCP Menu`、`Project Skills Menu`、`Git Quick Menu`；不要再把这些写进 `Composer`。
12. side-chat 是内联嵌套线程，不是常规 pane 顶部 tab；主 pane tabs 会过滤 side-chat session。
13. 左侧 `Session Tree` 行尾动作当前是“归档”，不是“删除”；恢复 / 永久删除统一在 `Settings > Archived Sessions`。
14. `Appearance` 只负责主题与壳层可见性；调试色、调试线、trace 可见性统一归 `Developer Mode`。
15. 设置页一级分组统一使用 `Provider`、`Models`、`MCP`、`Appearance`、`Developer Mode`、`Archived Sessions`，不要再沿用旧的四分组命名。

## 5. 维护规则

1. 新增稳定界面区域、ARIA label、类名锚点或测试依赖时，先同步更新本文档，再改代码。
2. 编写测试时，优先使用本文档中的类名、ARIA label 或稳定文案定位。
3. 如果新增、删除或重命名了稳定界面区域，必须同步更新“开发期区域调试色规范”中的区域名与背景色映射。
4. 如果 `Workbench` 分割模型、pane tabs、pane drag/drop、右侧栏视图集合、`Preview` / `Files` / `Runtime` 任一面板结构、`Composer Utility Bar` 子项、权限模式切换、session 归档流程、settings 一级分组或 side-chat 内联结构发生变化，必须同步更新本文档。
5. 如果 `CanvasRegionTopMenu` 重新回到主壳层，或者 `Pane Tab Bar` 再次被全局顶部 tabs 取代，必须显式更新本文档中的“当前界面结构”和“术语约束”，避免旧叫法继续流入设计讨论与测试代码。
